  - heading "Navigate to header Workspace file map (what each file means)" [ref=e64] [level=2]:
    - link "Navigate to header" [ref=e65] [nth=2]:
      - /url: "#workspace-file-map-what-each-file-means"
    - text: Workspace file map (what each file means)
  - text: "These are the standard files OpenClaw expects inside the workspace:"
      - code: AGENTS.md
        - listitem: Operating instructions for the agent and how it should use memory.
        - listitem: Loaded at the start of every session.
        - listitem: Good place for rules, priorities, and “how to behave” details.
      - code: SOUL.md
        - listitem: Persona, tone, and boundaries.
        - listitem: Loaded every session.
      - code: USER.md
        - listitem: Who the user is and how to address them.
        - listitem: Loaded every session.
      - code: IDENTITY.md
        - listitem: The agent’s name, vibe, and emoji.
        - listitem: Created/updated during the bootstrap ritual.
      - code: TOOLS.md
        - listitem: Notes about your local tools and conventions.
        - listitem: Does not control tool availability; it is only guidance.
      - code: HEARTBEAT.md
        - listitem: Optional tiny checklist for heartbeat runs.
        - listitem: Keep it short to avoid token burn.
      - code: BOOT.md
        - listitem: Optional startup checklist executed on gateway restart when internal hooks are enabled.
        - listitem: Keep it short; use the message tool for outbound sends.
      - code: BOOTSTRAP.md
        - listitem: One-time first-run ritual.
        - listitem: Only created for a brand-new workspace.
        - listitem: Delete it after the ritual is complete.
      - code: memory/YYYY-MM-DD.md
        - listitem: Daily memory log (one file per day).
        - listitem: Recommended to read today + yesterday on session start.
      - code: MEMORY.md
      - text: (optional)
        - listitem: Curated long-term memory.
        - listitem: Only load in the main, private session (not shared/group contexts).
  - text: See
  - link "Memory" [ref=e66] [nth=1]:
    - /url: /concepts/memory
  - text: for the workflow and automatic memory flush.
      - code: skills/
      - text: (optional)
        - listitem: Workspace-specific skills.
        - listitem: Overrides managed/bundled skills when names collide.
      - code: canvas/
      - text: (optional)
          - text: Canvas UI files for node displays (for example
          - code: canvas/index.html
          - text: ).
  - text: If any bootstrap file is missing, OpenClaw injects a “missing file” marker into the session and continues. Large bootstrap files are truncated when injected; adjust the limit with
  - code: agents.defaults.bootstrapMaxChars
  - text: "(default: 20000)."
  - code: openclaw setup
  - text: can recreate missing defaults without overwriting existing files.
